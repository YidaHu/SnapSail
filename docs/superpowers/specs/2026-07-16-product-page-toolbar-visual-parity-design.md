# SnapSail 产品页工具栏视觉对齐设计

- 日期：2026-07-16
- 状态：用户已指定产品页样式并要求直接实施
- 参考：`https://www.yidahu.top/projects/snapsail` 首屏“截图标注工具示意”

## 目标

把产品页中更轻盈的截图标注工具栏视觉迁移到真实 macOS 截图选框中。在不删除工具、不改变点击行为、不增加二级流程的前提下，让真实工具栏具备相同的白色悬浮胶囊、统一图标热区、淡蓝选中态、清晰分组和实心蓝色完成按钮。

## 视觉参数

- 工具栏尺寸从 `700 × 66 pt` 调整为 `744 × 62 pt`，圆角为 `22 pt`，不做完全药丸化。
- 背景使用接近纯白的 `#FCFCFC`，边框使用低对比度中性灰，阴影减轻黑度并增加扩散范围。
- 所有按钮使用 `40 × 42 pt` 热区，按钮之间保持 `4 pt` 间距。
- 图标统一使用 20pt、medium 权重的 SF Symbols；保持现有工具顺序和 tooltip。
- 普通状态为深石墨图标；悬停为中性浅灰底；选中状态为 `#E7F1FF` 浅蓝底和系统蓝图标。
- 复制完成是唯一实心蓝底、白图标的主操作；取消始终使用红色图标，悬停时使用淡红底。
- 分隔线高度为 26pt，并在“标注/编辑”和“取消/输出”之间各提供 8pt 视觉留白。

## 图标映射

产品页是图标视觉的基准，真实工具栏使用最接近的 20pt medium SF Symbols：

| 操作 | SF Symbol |
| --- | --- |
| 矩形 / 圆形 | `rectangle` / `circle` |
| 直线 / 箭头 | `line.diagonal` / `arrow.up.right` |
| 画笔 / 马赛克 | `pencil` / `square.grid.3x3` |
| 文字 / 序号 | `character` / `1.circle` |
| 高亮 / 颜色 | `highlighter` / `square.inset.filled` |
| 撤销 / 重做 / 取消 | `arrow.uturn.backward` / `arrow.uturn.forward` / `xmark` |
| 滚动截图 | `rectangle.and.hand.point.up.left` |
| 下载 | `arrow.down.to.line` |
| 复制完成 | `checkmark` |

复制完成必须显示勾号，不能继续使用双文档页；下载和滚动截图也必须使用彼此可区分的图标。真实 App 比产品页多出的直线、高亮和重做功能继续保留，但遵循同一套线性图标语言。

## 交互与行为

- 保留矩形、椭圆、直线、箭头、画笔、马赛克、文字、序号、高亮、颜色、撤销、重做、取消、滚动截图、下载、复制完成全部功能。
- 工具按钮继续支持再次点击取消选中；颜色按钮继续循环颜色。
- 撤销和重做不可用时保持可辨识但降低透明度。
- 点击区域、tooltip、回调、快捷键和选框定位算法不改变。

## 实现边界

- `SnapSailStyle` 承担工具栏颜色与几何 token。
- `InlineCaptureToolbar` 只负责按 token 布局控件和分组。
- `InlineToolbarButton` 负责 normal、hover、selected、destructive、primary、disabled 六类状态。
- 为关键按钮增加稳定的 accessibility identifier，既支持自动化测试，也便于后续辅助功能完善。

## 验收标准

1. 真实工具栏与产品页示意具有一致的白色圆角容器、间距节奏和蓝色层级。
2. 标注工具选中后显示淡蓝底和蓝图标，而不是整块高饱和蓝。
3. 复制完成按钮为实心蓝底白图标，视觉优先级最高。
4. 取消为红色，撤销/重做 disabled 状态清晰。
5. 所有现有按钮和回调仍然存在，完整测试套件和生产构建通过。
6. 每个按钮加载的 SF Symbol 与图标映射表一致，复制完成显示白色勾号。
