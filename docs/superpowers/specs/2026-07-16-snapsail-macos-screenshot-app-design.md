# SnapSail macOS 截图应用设计

- 日期：2026-07-16
- 状态：已完成对话评审，待用户复核书面规格
- 产品名：SnapSail
- 品牌短句：Capture More. Scroll Less.
- 最低系统：macOS 13 Ventura
- 发行渠道：官网直接下载与 Mac App Store

## 1. 产品目标

SnapSail 是一款原生 macOS 菜单栏截图工具，面向欧美市场，提供区域截图、窗口截图、手动滚动长截图、轻量标注、贴图置顶、历史记录和灵活导出。

产品体验参考 Xnip 的公开截图工作流与功能边界，但不复制其名称、图标、视觉资产或实现代码。SnapSail 使用独立品牌、现代 macOS 界面和自有工程实现。

核心承诺是：普通截图足够快，滚动截图足够稳，失败时不丢失已经捕获的内容。

## 2. 已确认的产品决策

1. 产品采用完整主要能力范围，不只交付截图演示版。
2. 同时支持官网分发和 Mac App Store 分发。
3. 最低兼容 macOS 13。
4. 滚动截图采用用户手动向下滚动的交互，不控制其他 App 自动滚动。
5. App 常驻菜单栏，不显示常规主窗口；设置、历史和标注窗口按需打开。
6. 产品名称为 SnapSail，英文为默认界面语言，同时支持简体中文。
7. 技术架构采用 Swift/AppKit、SwiftUI、ScreenCaptureKit、Vision、Core Image 和 Accelerate 的原生组合。

## 3. 用户与使用场景

主要用户是需要频繁分享视觉信息的开发者、设计师、产品经理、客服与内容创作者。

核心场景包括：

- 快速截取屏幕任意区域并复制到剪贴板。
- 截取单个窗口或按住 Shift 选择多个窗口，保留系统风格阴影。
- 截取网页、代码、聊天记录和长文档中超出当前屏幕的内容。
- 用箭头、形状、文字、高亮、模糊和编号快速标注截图。
- 将截图固定为始终置顶的参考窗口。
- 从本地历史中找回最近截图并重新复制、标注或导出。

## 4. 范围

### 4.1 包含的能力

- 区域截图。
- 窗口截图与 Shift 多窗口截图。
- 可配置的窗口阴影。
- 延时截图。
- 手动向下滚动截图。
- 滚动过程实时长图预览。
- 滚动失配后的暂停、继续和保留当前结果。
- 箭头、矩形、圆形、直线、自由画笔、文字、高亮、模糊和自动编号。
- 标注撤销与重做。
- 贴图置顶、移动、缩放、透明度调整和关闭。
- PNG 与 JPEG 导出。
- JPEG 质量和 PNG 透明背景设置。
- 保存到文件、复制到剪贴板和完成后自动动作。
- 文件名规则与默认保存目录。
- 本地捕获历史与自动清理周期。
- 登录时启动、菜单栏图标、通知、声音和触控板反馈。
- 区域、窗口、滚动、贴图和历史的可配置全局快捷键。
- 英文与简体中文本地化。
- 官网签名公证构建与 Mac App Store 构建。

### 4.2 当前不包含的能力

- 屏幕录像与音频录制。
- OCR 或文字识别。
- 自动控制其他 App 滚动。
- 云同步、账户系统和团队协作。
- Windows、Linux、iOS 或 iPadOS 版本。
- 向上或横向滚动拼接。

这些能力不进入当前实现计划。后续是否增加，需要单独进行产品设计。

## 5. 交互设计

### 5.1 菜单栏入口

菜单栏菜单按以下顺序提供操作：

1. Capture Area。
2. Capture Window。
3. Scrolling Capture。
4. Capture History。
5. Settings。
6. About SnapSail。
7. Quit SnapSail。

每个主要捕获操作显示当前快捷键。App 默认不显示 Dock 图标。

### 5.2 选择浮层

启动捕获后，每个显示器显示一个无边框透明浮层。未选区域使用半透明遮罩，当前窗口或拖拽区域保持清晰并显示蓝色边框。

交互规则：

- 移动鼠标时自动命中并高亮所在窗口。
- 拖拽后进入自由区域选择。
- Space 在窗口选择和自由区域之间切换。
- Shift 在窗口模式下增加或移除多选窗口。
- 方向键进行单像素微调，按住 Shift 时进行十像素微调。
- Esc 随时取消。
- 选择区域显示像素尺寸。
- 完成工具条位于选区外侧，不进入截图内容。

工具条包含 Capture、Scrolling Capture、Copy 和 Cancel。普通截图默认为 Capture；滚动截图入口在同一工具条中保持可见。

### 5.3 普通截图完成行为

用户确认选择后，CaptureEngine 获取最终图像，并按设置执行以下一种默认动作：

- 打开轻量标注窗口。
- 直接复制到剪贴板。
- 直接保存到默认目录。
- 同时复制并保存。

无论默认动作是什么，成功捕获均可写入本地历史；用户可以关闭历史记录。

### 5.4 滚动截图

用户选定只包含可滚动内容的区域后，点击 Scrolling Capture：

1. SnapSail 启动连续采帧，并在选区旁显示实时长图预览。
2. 用户用触控板、鼠标滚轮或键盘缓慢向下滚动。
3. 预览显示已捕获高度和当前匹配状态。
4. 匹配稳定时，新增内容连续追加。
5. 滚动过快、横向偏移过大或找不到可靠重叠时，捕获自动暂停。
6. 暂停时已拼接内容保持不变，界面提示用户减速后继续或直接完成。
7. 用户点击 Finish 后，当前长图进入与普通截图相同的标注和导出流程。
8. 用户点击 Cancel 时放弃当前会话，不写入历史。

滚动捕获只承诺静态或低动态内容。视频、持续动画、视差背景和大面积动态元素可能导致暂停，界面需要明确说明原因。

### 5.5 标注编辑器

标注编辑器使用独立文档模型，图像与标注对象分离保存，只有复制或导出时才进行最终合成。

工具包括：

- Arrow。
- Line。
- Rectangle。
- Ellipse。
- Pen。
- Text。
- Highlight。
- Blur。
- Step Number。

编辑器支持颜色、线宽、字体大小、模糊强度、撤销、重做、缩放和画布边缘调整。主操作为 Pin、Copy 和 Save。

### 5.6 贴图置顶

贴图窗口无标题栏、始终置顶，可拖动、缩放、调节透明度并通过快捷操作关闭。多张贴图可以同时存在。贴图不会自动写回原始截图。

### 5.7 设置中心

设置使用 SwiftUI 构建，包含以下分类：

- General：登录启动、菜单栏图标、通知、声音、触觉反馈、默认完成动作和语言。
- Capture：窗口阴影、多窗口、鼠标指针、选区放大镜和延时截图。
- Scrolling：预览位置、采帧策略、最大长图高度和失配提示。
- Annotation：工具顺序、默认颜色、线宽、字体和模糊强度。
- Export：PNG/JPEG、JPEG 质量、透明背景、命名规则、保存路径和复制后行为。
- Shortcuts：所有全局快捷键、冲突检测和恢复默认。
- SnapSail Pro：展示购买或许可证状态。

初始 Beta 不对功能设置付费墙。购买模块保留渠道适配边界，但具体 SKU、价格和 Pro 功能划分属于发布配置，不阻塞核心产品实现。

## 6. 技术架构

### 6.1 原生混合架构

AppKit 负责系统级窗口行为，包括菜单栏、透明选择浮层、多显示器窗口、贴图窗口和需要精确事件处理的交互。

SwiftUI 负责偏好设置、历史浏览、购买状态和适合声明式布局的普通界面。

ScreenCaptureKit 负责枚举显示器、App 和窗口，并提供高性能单帧或连续帧捕获。Vision 负责相邻帧图像配准，Core Image 与 Accelerate 负责裁剪、像素验证和最终合成。

参考资料：

- [Xnip 公开功能说明](https://xnipapp.com/)
- [Xnip 滚动截图说明](https://xnipapp.com/scrolling-capture/)
- [Apple ScreenCaptureKit](https://developer.apple.com/documentation/screencapturekit)
- [Apple Vision 图像配准](https://developer.apple.com/documentation/vision/vntranslationalimageregistrationrequest)
- [Apple NSStatusItem](https://developer.apple.com/documentation/appkit/nsstatusitem)

### 6.2 模块边界

- `AppCoordinator`：应用生命周期、菜单栏命令、窗口打开和服务装配。
- `MenuBarController`：NSStatusItem 与菜单状态。
- `HotkeyManager`：全局快捷键注册、冲突检测和触发分发。
- `PermissionCenter`：屏幕录制权限状态、系统设置跳转和权限恢复。
- `SelectionOverlayController`：每个显示器的选择浮层与跨屏坐标统一。
- `WindowHitTester`：可捕获窗口列表、鼠标命中和多窗口选择。
- `CaptureEngine`：ScreenCaptureKit 的单帧与连续帧接口。
- `ScrollCaptureSession`：滚动捕获状态机和串行帧处理。
- `FrameMatcher`：位移估计、重叠检测、置信度和异常分类。
- `ImageComposer`：分块长图存储、追加、预览和最终合成。
- `AnnotationDocument`：非破坏式标注数据与撤销栈。
- `AnnotationRenderer`：预览与最终图像渲染。
- `PinWindowController`：置顶贴图窗口生命周期。
- `CaptureHistoryRepository`：本地历史索引、缩略图和清理策略。
- `PreferencesStore`：用户设置持久化。
- `ExportService`：文件命名、PNG/JPEG 编码、剪贴板和保存面板。
- `PurchaseProviding`：StoreKit 与官网许可证的统一接口。

每个模块通过协议暴露最小接口。截图引擎、帧源、历史仓库、导出服务和购买服务都可注入测试替身。

## 7. 滚动拼接算法

### 7.1 帧采集

连续捕获只包含用户选定的矩形区域。捕获输出按显示器像素比例转换，所有内部合成坐标使用像素而不是 AppKit point。

帧到达后先计算轻量变化指标。变化低于阈值的帧直接丢弃，避免对静止画面重复执行配准。处理队列保持有界，只保留最新待处理帧，避免滚动过快时积压。

### 7.2 位移与重叠识别

FrameMatcher 执行两阶段匹配：

1. 使用 Vision 平移图像配准估计主要位移。
2. 在候选重叠带内执行局部像素相关性验证，并计算置信度。

只有满足以下条件的帧才允许追加：

- 主要位移方向向下。
- 横向偏移不超过容忍阈值。
- 重叠高度处于合理范围。
- 像素验证置信度超过阈值。
- 新增内容高度大于最小追加值。

符合条件时，ImageComposer 裁掉重复重叠，仅追加当前帧的新底部内容。

### 7.3 内存与临时数据

长图不在整个会话期间维持单个巨大位图。ImageComposer 使用固定高度图块和磁盘临时缓存：

- 内存中只保留最近图块、匹配所需尾部区域和缩略预览。
- 完成时流式合成最终 PNG 或 JPEG。
- 用户可以配置最大捕获高度，默认上限为 60,000 像素。
- 达到上限时自动完成并提示原因。

临时文件写入 Application Support 下的会话目录，并使用原子元数据更新。正常完成、取消或下次启动恢复清理时删除无用临时文件。

### 7.4 失配处理

以下情况进入暂停状态：

- 找不到可靠重叠。
- 横向位移超过阈值。
- 用户向上滚动。
- 内容变化主要来自动画而不是滚动。
- 帧处理持续落后。

暂停不会修改当前长图。后续帧重新与最后一个已接受帧匹配；恢复成功后继续追加。连续多次恢复失败时，界面建议完成当前结果或重新选择区域。

## 8. 数据与隐私

SnapSail 不要求账户，不上传截图、标注、历史或使用数据。

本地数据分为：

- `UserDefaults`：轻量偏好设置和快捷键。
- `Application Support/SnapSail/History`：历史索引、缩略图和可选原图。
- `Application Support/SnapSail/Sessions`：滚动截图临时图块。
- 用户选择的导出目录：最终文件。

历史记录默认开启，保留 30 天。用户可以关闭历史、改为永久保留或选择 1、7、30、90 天自动清理。关闭历史后，只有用户明确保存的文件会保留。

## 9. 权限与双渠道发行

首次执行截图时请求屏幕录制权限，不在首次启动时无上下文弹出权限提示。若系统要求重新启动 App 才能生效，SnapSail 解释原因并提供 Relaunch 按钮。

官网版和 App Store 版共用源码与数据模型，使用独立 Xcode Scheme 和配置文件：

- `SnapSail-Direct`：Developer ID 签名、公证与官网许可证适配器。
- `SnapSail-AppStore`：Mac App Store 签名、沙盒与 StoreKit 适配器。

两个版本使用相同 bundle 数据迁移规则。渠道差异只能进入构建配置和 `PurchaseProviding` 实现，不能渗透到截图、拼接或标注模块。

## 10. 错误处理

- 屏幕录制未授权：解释用途，提供打开系统设置和重新检测操作。
- 快捷键冲突：阻止保存冲突组合，标出冲突项，并保留旧快捷键。
- 滚动过快或无法匹配：暂停并保留当前结果，允许继续、完成或取消。
- 显示器断开或缩放变化：停止捕获，保存可用的已拼接内容。
- 内存压力：释放预览缓存并降低预览刷新率，不丢弃已落盘图块。
- 磁盘空间不足：停止追加并允许导出当前可用结果到其他位置。
- 导出失败：保留原始捕获与标注文档，允许重新导出。
- 历史索引损坏：从现有文件重建最小索引，将不可解析条目隔离。
- App 异常退出：下次启动清理不完整临时会话，不删除已完成历史。
- 购买服务离线：保留最后一次有效授权状态，并提供恢复购买或重新验证入口。

## 11. 测试设计

### 11.1 单元测试

- AppKit point、ScreenCaptureKit frame 与像素坐标转换。
- Retina、非 Retina 与不同屏幕原点组合。
- 窗口命中与多窗口选择集合。
- 快捷键编码、冲突检测与恢复默认。
- 文件名模板、扩展名和重复文件处理。
- 历史保留周期与清理策略。
- FrameMatcher 的位移、重叠和置信度计算。
- ImageComposer 的去重追加、图块边界和最大高度。
- 标注命令的撤销与重做。

### 11.2 合成帧与像素回归测试

测试资源由已知长图按不同偏移裁成帧序列，覆盖：

- 匀速向下滚动。
- 不等步长滚动。
- 重复纹理。
- 固定顶部栏。
- 轻微横向偏移。
- 滚动过快导致低重叠。
- 中途停顿与恢复。
- 向上滚动。

输出长图与黄金图进行尺寸、像素差异和接缝区域回归比较。

### 11.3 集成测试

使用可注入的假帧源验证完整状态机：idle、selecting、capturing、paused、finishing、completed、cancelled 和 failed。

验证权限变化、显示器断开、磁盘不足、导出失败和 App 重启清理。

### 11.4 手工兼容性矩阵

- 单 Retina 屏幕。
- Retina 主屏加外接非 Retina 屏。
- 显示器位于主屏左侧、右侧和上方。
- 浅色与深色模式。
- Safari、Chrome、Slack 或同类聊天窗口、VS Code 或同类编辑器。
- 区域截图、单窗口、多窗口、滚动、标注、贴图、复制和保存。
- 官网签名公证构建与 App Store 沙盒构建。

## 12. 验收标准

1. 区域截图在所有测试显示器排列中与可见选区一致，不出现比例或坐标偏移。
2. 单窗口和多窗口捕获内容正确，窗口阴影设置生效。
3. 普通截图可以在确认选区后的一个动作内复制、保存或进入标注。
4. Safari、Chrome、聊天窗口和代码编辑器中的静态长内容可完成手动向下滚动拼接。
5. 滚动失配时自动暂停且不损坏已拼接内容，恢复后不会重复追加已有区域。
6. 标注工具支持撤销与重做，预览、复制和最终导出结果一致。
7. 贴图窗口可以同时存在多张，并保持置顶、移动和缩放行为正确。
8. 权限拒绝、快捷键冲突、显示器变化、磁盘不足和导出失败均有明确且可恢复的用户路径。
9. 所有截图和历史数据默认只在本机处理。
10. 官网版和 App Store 版通过各自的签名、权限与安装验证。

## 13. 分阶段交付

### 阶段一：截图内核

完成菜单栏、快捷键、权限、区域选择、窗口与多窗口捕获、剪贴板、PNG/JPEG 保存和核心设置。

### 阶段二：滚动长图

完成连续帧捕获、匹配、分块合成、实时预览、暂停恢复、长图导出和合成帧回归测试。

### 阶段三：完整工作流

完成标注编辑器、贴图置顶、历史记录、全部偏好设置和中英文界面。

### 阶段四：双渠道发行

完成 StoreKit 与官网许可证适配边界、沙盒检查、Developer ID 签名、公证、App Store 构建和发布前兼容性矩阵。

每一阶段必须通过自身测试和上一阶段回归测试后才能进入下一阶段。

## 14. 成功定义

SnapSail 的成功不是只生成一个可运行窗口，而是形成一个可持续发行的原生截图产品：普通截图快速可靠，滚动截图失败可恢复，完整工作流无需离开 App，且同一套核心能力能够安全地交付到官网和 Mac App Store。
